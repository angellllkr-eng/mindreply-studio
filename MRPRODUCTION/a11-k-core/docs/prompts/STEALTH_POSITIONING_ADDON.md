# Stealth Positioning Add-On

Apply this to all public surfaces.

Do not make the product look like a raw AI/backend/internal dashboard.
Do not expose internal architecture.
Do not publicly show secrets, private logs, repo paths, webhook URLs, provider keys, or admin/owner systems.
Do not fake active automation.

Make public copy calm, premium, and understandable.

AUREL:
Use AUREL as the main premium private execution layer.
Public line:
“AUREL is a private execution layer for decisions, workflows, actions, and protected execution.”

A11-K:
Use A11-K as the quiet engine/stamp.
Public line:
“Powered by A11-K.”
Do not overuse the A11-K mark.

MindReply:
Keep MindReply public and simple.
Public line:
“Finish your website. Clear your response overload.”

Brushworks:
Keep Brushworks creative and visual.
Public line:
“Built with Brushworks. Powered by A11-K.”

Rename public labels:
- Shadow Company -> Operating Twin
- DecisionTwin -> Decision Preview
- WorkflowTwin -> Workflow Status
- ModelTwin -> AI Model Status
- Autonomy Levels -> Action Safety Levels
- Escalation Rules -> When to Notify Angel
- n8n -> workflow automation
- webhook -> workflow connection
- model router -> routing layer
- rate limit -> usage limit

Every public page must answer:
1. What is this?
2. Why does it matter?
3. What can I do here?
4. What is working?
5. What is waiting or blocked?
6. What should I do next?

Private /command can show more operational detail, but must remain protected and noindexed.
