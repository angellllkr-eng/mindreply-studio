# A11-K Identity System

## Hierarchy

1. **A11-K** — engine core, operating system, command layer, and quiet certification stamp.
2. **MindReply** — public commercial brand.
3. **Brushworks** — warmer visual/build system and creative execution layer.
4. **Tools** — distinct product modules: SQL Studio, Code Lens, SDR Agent, Regex Forge, Code Tutor, L402 Skills.
5. **Labs** — experiments: AI Arena, Aether-X, Kratos-S.
6. **Future brands** — ideas and vault concepts: Aurel, Meridian, BRIXO, Yacht/Marina.

## A11-K mark

- Use the A11 geometry and a restrained violet/lilac signal.
- Keep the mark small, sharp, and consistent.
- Pair with one quiet phrase: `Powered by A11-K`, `Secured by A11-K`, `Routed through A11-K`, or `Decision preview by A11-K`.
- Use the stamp as a certification, not a takeover.
- A11-K root and command are darker, more authoritative, and more technical than every child brand.

## Distinction rules

- Do not clone A11-K UI into MindReply, Brushworks, or tools.
- Child brands keep their own color, voice, and layout.
- A11-K is the shared governance signal.
- Never imply a child product is live or connected without verified evidence.
- Never expose command routes, secrets, customer data, or internal provider values on public pages.

## Product language

- Root: “The engine behind everything you are building.”
- Command: “See the signal. Choose the move.”
- Stamp: “Powered by A11-K.”
- MindReply: “Real-Time Event Tech, One Text Away.” only after public contact/service verification.
- Brushworks: “The visual build layer for fast, premium AI-native products.”
